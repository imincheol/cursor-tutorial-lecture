## Debug Mode 실습

### 사용법

프롬프트에 "debug this" 또는 "디버그해줘"

### 🔥 실전 Debug 시나리오들

#### 1. 간헐적 API 실패
```
"사용자 목록 API가 가끔 500 에러를 반환해.

재현:
1. 사용자 관리 페이지 접속
2. 검색어 입력 후 검색
3. 5번 중 1번 정도 500 에러

브라우저 콘솔에 특별한 에러 없음.
디버그해줘."
```

#### 2. 무한 로딩 상태
```
"상품 상세 페이지가 로딩 중으로 계속 멈춰있어.

재현:
1. 상품 목록에서 아무 상품 클릭
2. 상세 페이지로 이동
3. 로딩 스피너만 계속 돌고 데이터 안 나옴

네트워크 탭에는 API 호출이 없음.
디버그해줘."
```

#### 3. 폼 제출 후 이상 동작
```
"회원가입 폼 제출 후 성공 메시지가 안 보여.

재현:
1. 회원가입 폼 작성
2. 제출 버튼 클릭
3. API는 200 응답하는데 성공 메시지 미표시
4. 페이지도 안 넘어감

디버그해줘."
```

#### 4. 상태 관리 버그
```
"장바구니에 상품 추가했는데 수량이 안 올라가.

재현:
1. 상품 페이지에서 "장바구니 추가" 클릭
2. 헤더의 장바구니 아이콘 수량은 그대로
3. 장바구니 페이지 가서 확인하면 상품은 들어있음

상태 업데이트가 안 되는 것 같아.
디버그해줘."
```

#### 5. 이벤트 핸들러 문제
```
"버튼 클릭해도 아무 반응 없어.

재현:
1. 설정 페이지로 이동
2. "알림 설정" 토글 버튼 클릭
3. 시각적 변화 없음, 콘솔 에러도 없음

클릭 이벤트가 안 걸리는 것 같아.
디버그해줘."
```

### Agent 작업 흐름

```
1. 관련 파일 분석
2. 의심 지점에 로그 삽입
3. "버그 재현해주세요" 요청
4. 로그 분석 → 원인 파악
5. 수정 제안
```

### Debug Mode 팁

```
🎯 효과적인 디버그 요청:
- 구체적인 재현 단계 설명
- 기대 동작 vs 실제 동작
- 발생 빈도 (항상/가끔/특정 조건)
- 브라우저 콘솔/네트워크 상태
- "이 함수가 언제 호출되는지 확인해줘"
- "이 변수 값이 어떻게 변하는지 추적해줘"
```

## Visual Editor 실습

### 열기

```
Cmd+Shift+P → "Browser: Open Preview"
```

### 앱 실행

```bash
npm run dev
```

### 요소 선택

```
Browser 패널 상단 → 선택 도구 (커서 아이콘)
→ 수정할 요소 클릭
```

### 🔥 실전 UI 수정 시나리오들

#### 1. 버튼 스타일링
```
요소 선택 후:
"이 버튼을 더 현대적으로 바꿔줘 - 둥근 모서리, 그림자 효과, 호버 애니메이션 추가"
```

#### 2. 카드 컴포넌트 개선
```
요소 선택 후:
"이 카드를 더 예쁘게 디자인해줘 - 테두리, 그림자, 내부 패딩 최적화"
```

#### 3. 폼 요소 스타일링
```
요소 선택 후:
"이 입력 필드를 더 사용자 친화적으로 - 포커스 효과, 에러 상태 스타일, 플레이스홀더 개선"
```

#### 4. 레이아웃 조정
```
요소 선택 후:
"이 섹션의 레이아웃을 더 균형 있게 - 그리드 시스템 적용, 간격 최적화"
```

#### 5. 반응형 디자인
```
요소 선택 후:
"모바일에서도 잘 보이게 반응형으로 수정해줘 - 미디어 쿼리 추가, 플렉시블 레이아웃"
```

#### 6. 색상 테마 적용
```
요소 선택 후:
"다크 모드 지원하도록 색상 변수로 바꿔줘 - CSS 커스텀 프로퍼티 사용"
```

### 고급 Visual Editor 기능

#### Component Tree 탐색
```
Browser 패널 → Component Tree 탭
→ React/Vue 컴포넌트 계층 구조 확인
→ 특정 컴포넌트 선택하여 수정
```

#### 요소별 스타일 검사
```
요소 선택 → Styles 패널
→ 현재 적용된 CSS 확인
→ "이 스타일을 더 일관성 있게 개선해줘"
```

#### 상태별 스타일링
```
요소 선택 후:
"호버/포커스/액티브 상태에 맞는 스타일 추가해줘"
```

### Visual Editor 팁

```
🎨 효과적인 요청:
- 구체적인 시각적 결과물 설명
- "더 현대적으로", "더 깔끔하게" 같은 추상적 표현
- 색상, 크기, 간격 등 구체적 수치
- "참고 디자인: [링크]" 형태로 레퍼런스 제공
- "모바일에서도 잘 보이게"
- "접근성 고려해서 (WCAG)"
```

## IDE Multi-Agent 주의점

```
⚠️ 같은 브랜치에서 작업

안전한 사용:
- 다른 파일 작업
- 순차적 작업
- 질문/분석만 (수정 X)
```

## 실습 과제

### 기본 실습 (필수)
1. [ ] Debug Mode로 버그 분석 시도 (간헐적 API 실패 시나리오)
2. [ ] Visual Editor 열기
3. [ ] 요소 선택 → Agent로 수정 (버튼 스타일링)
4. [ ] Component Tree 확인

### 🔥 보너스 실습 (선택)

#### 초급 보너스
5. [ ] **다양한 버그 시나리오 Debug**
   - 무한 로딩, 폼 제출, 상태 관리 버그 각각 시도
   - Debug Mode의 자동 로그 삽입 체험

6. [ ] **복합 UI 수정**
   - 카드 컴포넌트 + 폼 요소 동시 수정
   - "이 두 요소의 디자인을 통일해줘"

7. [ ] **반응형 디자인 적용**
   - 모바일/데스크톱에서 다른 모습 구현
   - 미디어 쿼리 자동 생성 확인

#### 중급 보너스
8. [ ] **색상 테마 시스템 구축**
   - CSS 변수 활용한 다크/라이트 모드
   - "전체 앱에 일관된 색상 테마 적용해줘"

9. [ ] **컴포넌트 계층 분석**
   - Component Tree로 복잡한 컴포넌트 구조 파악
   - 상태가 어디서 관리되는지 추적

10. [ ] **접근성 개선**
    - WCAG 가이드라인 준수하도록 수정
    - "스크린 리더 사용자를 위해 개선해줘"

#### 고급 보너스
11. [ ] **실전 프로젝트 적용**
    - 실제 작업 중인 프로젝트에 Visual Editor 적용
    - 디자인 시스템 구축

12. [ ] **Debug + Visual 결합**
    - 버그 발견 → Debug Mode → Visual Editor로 수정
    - 완전한 문제 해결 워크플로우 체험

13. [ ] **성능 최적화**
    - Visual Editor로 렌더링 성능 개선
    - "이 컴포넌트 리렌더링을 최적화해줘"

### 📊 실습 완료도 체크

**보너스 실습 (5/9) 완료 시:**

- 🔥 실무에서 즉시 활용 가능한 수준
