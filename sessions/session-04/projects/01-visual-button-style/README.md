## 🎨 실습 시나리오

페이지에 여러 개의 버튼이 있습니다. 특정 버튼의 스타일을 수정해야 합니다.

**기존 방식 (말로 설명)**:
```
"왼쪽 위에 있는 파란색 버튼을 빨간색으로 바꿔줘"
→ Agent가 어떤 버튼인지 추측
→ 틀리면 다시 설명
```

**Visual Editor 방식**:
```
1. Visual Editor 활성화
2. 버튼 클릭
3. "이 버튼을 빨간색으로 바꿔줘"
→ 정확히 선택된 버튼만 수정
```

---

## 🚀 실습 단계

### Step 1: 페이지 열기 (2분)

1. `index.html` 파일을 브라우저에서 열기
2. 여러 개의 버튼이 보이는지 확인

---

### Step 2: 일반 모드로 시도 (5분)

Cursor Agent에게 다음과 같이 요청하세요:

```
"Primary" 버튼의 배경색을 빨간색으로 바꿔줘
```

**문제점**:
- Agent가 어떤 버튼인지 정확히 모름
- 여러 버튼 중 하나를 추측해서 수정
- 틀리면 다시 설명해야 함

---

### Step 3: Visual Editor 활성화 (5분)

1. Cursor Agent 창에서 **Visual Editor** 버튼 클릭
2. 브라우저가 자동으로 열림 (또는 URL 입력)
3. 수정하고 싶은 버튼을 **직접 클릭**
4. Agent에게 요청:

```
이 버튼을 빨간색으로 바꿔줘
```

**결과**:
- ✅ 정확히 선택된 버튼만 수정됨
- ✅ 말로 설명할 필요 없음
- ✅ 한 번에 완료

---

### Step 4: 다양한 스타일 수정 (3분)

Visual Editor로 다음 작업들을 시도하세요:

1. **크기 변경**:
   - 버튼 클릭
   - "이 버튼을 더 크게 만들어줘"

2. **둥근 모서리**:
   - 버튼 클릭
   - "이 버튼을 완전히 둥글게 만들어줘"

3. **그림자 추가**:
   - 버튼 클릭
   - "이 버튼에 그림자 효과 추가해줘"

## 💡 핵심 개념

### 일반 모드 vs Visual Editor

| 방식       | 일반 모드        | Visual Editor    |
| ---------- | ---------------- | ---------------- |
| **선택**   | 말로 설명        | 클릭으로 선택    |
| **정확도** | 추측 필요        | 100% 정확        |
| **소통**   | 여러 번 설명     | 한 번에 완료     |
| **시간**   | 오래 걸림        | 빠름             |

### Visual Editor의 장점

1. **정확한 선택**: 말로 설명할 필요 없음
2. **빠른 수정**: 한 번에 원하는 요소 수정
3. **직관적**: 보이는 대로 선택

## 🎨 수정 예시

### Before (일반 모드)

```
당신: "파란색 버튼을 빨간색으로 바꿔줘"
Agent: [첫 번째 버튼 수정]
당신: "아니, 그 버튼 말고 옆에 있는 버튼!"
Agent: [두 번째 버튼 수정]
당신: "아니, 그것도 아니고..."
```

### After (Visual Editor)

```
당신: [버튼 클릭]
당신: "이 버튼을 빨간색으로 바꿔줘"
Agent: [정확히 선택된 버튼 수정]
당신: "완벽해!"
```
