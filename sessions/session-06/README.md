- [ ] HTML/CSS 품질을 자동으로 검증
- [ ] 다중 테마 검증
- [ ] 컴포넌트 라이브러리 관리

**소요 시간**: 50분

---

## 📚 교재 내용

### 1. HTML/CSS 품질 검증 (20분)

#### 순차 처리 (기존)
```
HTML 시맨틱 (10분)
  ↓
접근성 검증 (10분)
  ↓
SEO 검증 (10분)
  ↓
CSS 품질 (15분)

총 45분
```

#### 병렬 처리 (개선)
```
Worktree 1: HTML 시맨틱 (10분)
Worktree 2: 접근성 검증 (10분)
Worktree 3: SEO 검증 (10분)
Worktree 4: CSS 품질 (15분)

총 18분 (60% 단축!)
```

#### 검증 항목

**HTML 시맨틱**:
- 적절한 태그 사용
- 구조적 마크업
- 메타 정보
- 폼 요소

**접근성 검증**:
- ARIA 속성
- 대체 텍스트
- 키보드 접근성
- 색상 대비

**SEO 검증**:
- 메타 태그
- 구조화된 데이터
- 시맨틱 HTML
- 성능 최적화

**CSS 품질**:
- 구조 및 조직
- 성능 최적화
- 네이밍 컨벤션
- 반응형 디자인

### 2. 다중 테마 검증 (15분)

#### 테마 종류
- 라이트 테마
- 다크 테마
- 고대비 테마
- 커스텀 테마

#### 병렬 검증
```bash
worktree-light/      # 라이트 테마
worktree-dark/       # 다크 테마
worktree-contrast/   # 고대비 테마
```

#### 검증 항목
- 색상 일관성
- 가독성
- 접근성
- 전환 애니메이션

### 3. 컴포넌트 라이브러리 관리 (15분)

#### 검증 영역
- 컴포넌트 품질
- 디자인 시스템 준수
- 아이콘 시스템
- 타이포그래피
- 간격 시스템
- 애니메이션

## 🚀 실습 프로젝트

### Project 1: html-css-quality
**목표**: HTML/CSS 품질 병렬 검증

**실습 내용**:
1. 4개 Worktree 생성
2. 각각 독립적인 검증 실행
   - HTML 시맨틱
   - 접근성
   - SEO
   - CSS 품질
3. 이슈 수집
4. 개선 제안

**소요 시간**: 15분

**자동화 스크립트**: `check-html-css.sh`

---

### Project 2: theme-validation
**목표**: 다중 테마 병렬 검증

**실습 내용**:
1. 3개 Worktree 생성 (라이트, 다크, 고대비)
2. 각 테마별 검증
3. 일관성 체크
4. 접근성 확인

**소요 시간**: 15분

**자동화 스크립트**: `check-themes.sh`

---

### Project 3: component-library
**목표**: 컴포넌트 라이브러리 품질 관리

**실습 내용**:
1. 컴포넌트별 Worktree 생성
2. 각 컴포넌트 품질 검증
3. 디자인 시스템 준수 확인
4. 문서화 상태 확인

**소요 시간**: 20분
