   → 자동화/스크립트용

   → 컨텍스트 없음

### 3. Shell Mode 명확화

#### curriculum/03-session.md에 추가

**"Shell Mode란?" 섹션**:
```markdown
### Shell Mode란?

**자연어를 터미널 명령어로 변환해주는 특수 모드**

핵심 특징:
- ✅ 명령어만 생성: 코드 파일 직접 수정 안 함
- ✅ 실행 전 확인: 명령어 보여주고 실행 여부 물어봄
- ✅ 안전함: 위험한 명령은 경고
- ✅ 학습 효과: 생성된 명령어 보며 배움
```

**"Shell Mode vs 대화형 Agent" 비교표**:

| 항목 | 대화형 Agent | Shell Mode |
|------|-------------|-----------|
| **목적** | 코드 작성/수정 | 명령어 생성 |
| **파일 수정** | ✅ 가능 | ❌ 불가 |
| **명령 실행** | ⚠️ 제한적 | ✅ 주 기능 |
| **실행 확인** | - | ✅ 물어봄 |
| **예시** | "리팩토링해줘" | "파일 찾아줘" |

**언제 Shell Mode?**
- ✅ 파일 찾기/정렬/필터링
- ✅ 프로세스 관리 (포트, 메모리)
- ✅ Git 명령어
- ✅ 시스템 모니터링
- ✅ Docker/npm 명령어

**언제 대화형 Agent?**
- ✅ 코드 작성/수정
- ✅ 리팩토링
- ✅ 버그 수정
- ✅ 코드 분석

### 기억할 것

1. **대화형이 기본**: 모르면 `cursor agent`
2. **Shell Mode는 명령어 전용**: 코드 수정 안 함
3. **비대화형은 자동화용**: 대화 필요하면 대화형 사용
