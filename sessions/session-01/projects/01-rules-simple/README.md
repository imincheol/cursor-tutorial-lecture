## 🚀 실습 단계

### Step 1: 기본 응답 확인

먼저 Rules 없이 AI가 어떻게 답변하는지 확인해봅시다.

Cursor Agent를 열고 `test.js` 파일을 선택한 후 다음을 요청하세요:

```
test.js의 greet 함수에 주석을 추가하고, numbers 배열을 map으로 2배로 만드는 코드를 추가해줘
```

**결과 확인**:
- 아마도 영어 주석이 포함될 것입니다
- 일반적인 코드 스타일로 작성될 것입니다

---

### Step 2: Rules 적용 - 한국어 응답

이제 Rules를 적용해봅시다.

#### 2-1. Rules 파일 생성

프로젝트 루트에 `.cursorrules` 파일을 만들고, **`sample-rules-1-korean.md`** 파일의 내용을 복사해서 붙여넣으세요:

```
# 전역 규칙

- 모든 답변은 한국어로 작성하세요
- 주석도 모두 한국어로 작성하세요
- 설명할 때 이모지를 사용하세요
```

**파일 위치**: 프로젝트 루트 (예: `/Users/user/_/workspace/nts/study/cursor-tutorial/.cursorrules`)

#### 2-2. 변경된 응답 확인

`test.js` 파일을 원래대로 되돌린 후, 같은 질문을 다시 해보세요:

```
test.js의 greet 함수에 주석을 추가하고, numbers 배열을 map으로 2배로 만드는 코드를 추가해줘
```

**결과 확인**:
- ✅ 주석이 한국어로 작성되었나요?
- ✅ 설명에 이모지가 포함되었나요?
- ✅ 전체 답변이 한국어인가요?

---

### Step 3: Rules 변경 - 코드 스타일

이번에는 코드 스타일을 변경해봅시다.

#### 3-1. Rules 파일 수정

`.cursorrules` 파일의 내용을 **`sample-rules-2-style.md`** 파일의 내용으로 교체하세요:

```
# 코드 스타일 규칙

- 모든 함수에 JSDoc 주석을 추가하세요
- 변수명은 camelCase를 사용하세요
- 화살표 함수를 사용하세요
- 세미콜론을 항상 사용하세요
```

#### 3-2. 변경된 스타일 확인

`test.js` 파일을 원래대로 되돌린 후, 다음을 요청하세요:

```
test.js의 greet 함수를 개선하고, numbers 배열의 합계를 구하는 함수를 추가해줘
```

**결과 확인**:
- ✅ JSDoc 주석이 있나요?
- ✅ 화살표 함수를 사용했나요?
- ✅ 세미콜론이 있나요?

---

### Step 4: Rules의 한계 확인

이제 Rules의 한계를 경험해봅시다.

#### 4-1. 엄격한 Rules 적용

`.cursorrules` 파일의 내용을 **`sample-rules-3-strict.md`** 파일의 내용으로 교체하세요:

```
# 엄격한 규칙

- 절대로 console.log를 사용하지 마세요
- 절대로 var 키워드를 사용하지 마세요
- 절대로 any 타입을 사용하지 마세요
```

#### 4-2. 규칙 위반 요청

`test.js` 파일을 선택한 후, 다음을 요청하세요:

```
test.js에 console.log로 numbers 배열을 출력하는 코드를 추가해줘
```

**결과**:
- ❓ console.log를 사용했나요?
- ❓ 규칙을 무시했나요?

**깨달음**: 
Rules는 **프롬프트**일 뿐입니다. AI가 사용자의 명시적인 요청을 우선시하여 규칙을 무시할 수 있습니다. "절대로 하지 마세요"라고 해도 강제할 수 없습니다.

→ **3장에서 배울 Hooks는 코드로 100% 강제할 수 있습니다!**

## 💡 핵심 개념

### 1. Rules는 프롬프트

```
.cursorrules 파일 = AI에게 보내는 시스템 프롬프트

"이렇게 해주세요" (요청)
≠ "이렇게 해야 합니다" (강제)
```

### 2. Rules의 장점

- ✅ 일관된 코드 스타일 유지
- ✅ 프로젝트 컨벤션 자동 적용
- ✅ 반복적인 설명 불필요

### 3. Rules의 한계

- ❌ 가끔 무시됨
- ❌ 강제할 수 없음
- ❌ 위험한 작업도 막을 수 없음
- ❌ 사용자의 명시적 요청이 우선됨

### 4. Copilot과의 비교

| 항목 | Copilot | Cursor |
|-----|---------|--------|
| 파일명 | `.github/copilot-instructions.md` | `.cursorrules` |
| 위치 | `.github/` 폴더 안 | 프로젝트 루트 |
| 기능 | 동일 | 동일 |

## 📝 실습 팁

### 샘플 Rules 활용 방법

1. **복사**: `sample-rules-*.md` 파일의 내용을 복사
2. **붙여넣기**: 프로젝트 루트의 `.cursorrules` 파일에 붙여넣기
3. **테스트**: Agent에게 요청하고 결과 확인
4. **비교**: 다른 샘플로 교체하고 차이 확인

### test.js 파일 활용

- 간단한 파일이므로 빠르게 테스트 가능
- 원래 상태로 되돌리기 쉬움
- 다양한 요청으로 실험 가능
