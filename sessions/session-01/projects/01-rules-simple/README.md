# Project 1: Rules 간단 실습

> **질의응답으로 바로 확인하는 Rules 기능**

## 🎯 학습 목표

- [ ] `.cursorrules` 파일로 AI 응답 언어 변경
- [ ] Rules가 AI의 답변에 어떻게 영향을 주는지 확인
- [ ] Rules의 한계 경험

**예상 소요 시간**: 10분

---

## 🚀 실습 단계

### Step 1: 기본 응답 확인 (2분)

먼저 Rules 없이 AI가 어떻게 답변하는지 확인해봅시다.

Cursor Agent를 열고 다음을 질문하세요:

```
간단한 로그인 함수를 만들어줘
```

**결과 확인**:
- 아마도 영어 주석이나 설명이 포함될 것입니다
- 일반적인 코드 스타일로 작성될 것입니다

---

### Step 2: Rules로 언어 변경 (3분)

이제 프로젝트 루트에 `.cursorrules` 파일을 만들고 다음 내용을 작성하세요:

```
# 전역 규칙

- 모든 답변은 한국어로 작성하세요
- 주석도 모두 한국어로 작성하세요
- 설명할 때 이모지를 사용하세요
```

**파일 위치**: 프로젝트 루트 (예: `/Users/user/_/workspace/nts/study/cursor-tutorial/.cursorrules`)

---

### Step 3: 변경된 응답 확인 (2분)

같은 질문을 다시 해보세요:

```
간단한 로그인 함수를 만들어줘
```

**결과 확인**:
- ✅ 주석이 한국어로 작성되었나요?
- ✅ 설명에 이모지가 포함되었나요?
- ✅ 전체 답변이 한국어인가요?

---

### Step 4: 다양한 Rules 실험 (3분)

`.cursorrules` 파일을 다음과 같이 수정해보세요:

```
# 실험 1: 코드 스타일 변경

- 모든 함수에 JSDoc 주석을 추가하세요
- 변수명은 snake_case를 사용하세요
- 화살표 함수 대신 function 키워드를 사용하세요
```

다시 질문하세요:
```
사용자 정보를 가져오는 함수를 만들어줘
```

**결과 확인**:
- ✅ JSDoc 주석이 있나요?
- ✅ 변수명이 snake_case인가요? (예: `user_name`)
- ✅ `function` 키워드를 사용했나요?

---

### Step 5: Rules의 한계 확인 (2분)

이제 Rules의 한계를 경험해봅시다.

`.cursorrules` 파일에 다음을 추가하세요:

```
# 중요한 규칙

- 절대로 console.log를 사용하지 마세요
- 절대로 var 키워드를 사용하지 마세요
```

그리고 다음을 요청하세요:
```
디버깅을 위해 console.log를 사용해서 사용자 정보를 출력하는 함수를 만들어줘
```

**결과**:
- ❓ console.log를 사용했나요?
- ❓ 규칙을 무시했나요?

**깨달음**: 
Rules는 **프롬프트**일 뿐입니다. AI가 가끔 무시할 수 있습니다. "절대로 하지 마세요"라고 해도 강제할 수 없습니다.

→ **3장에서 배울 Hooks는 코드로 100% 강제할 수 있습니다!**

## 💡 핵심 개념

### 1. Rules는 프롬프트

```
.cursorrules 파일 = AI에게 보내는 시스템 프롬프트

"이렇게 해주세요" (요청)
≠ "이렇게 해야 합니다" (강제)
```

### 2. Rules의 장점

- ✅ 일관된 코드 스타일 유지
- ✅ 프로젝트 컨벤션 자동 적용
- ✅ 반복적인 설명 불필요

### 3. Rules의 한계

- ❌ 가끔 무시됨
- ❌ 강제할 수 없음
- ❌ 위험한 작업도 막을 수 없음

### 4. Copilot과의 비교

| 항목 | Copilot | Cursor |
|-----|---------|--------|
| 파일명 | `.github/copilot-instructions.md` | `.cursorrules` |
| 위치 | `.github/` 폴더 안 | 프로젝트 루트 |
| 기능 | 동일 | 동일 |
